# GitHub Commit & Push Workflow

現在の変更をステージングし、AIが生成した適切なコミットメッセージでコミットを行った後、現在のブランチをリモートリポジトリにプッシュします。

## ワークフロー

1. **変更の確認とステージング**
   - `git status` および `git diff` を使用して、現在の変更内容を把握します。
   - `run_shell_command` を使用して、すべての変更をステージングします。
     ```bash
     git add .
     ```

2. **コミットメッセージの生成**
   - 差分情報に基づき、以下の基準でコミットメッセージを生成します。
     - **形式**: Conventional Commits (feat, fix, docs, style, refactor, test, chore) に準拠。
     - **内容**: 変更の理由と内容を簡潔に説明。
     - **言語**: 日本語。

3. **コミットの実行**
   - 生成したメッセージを使用してコミットを実行します。
     ```bash
     git commit -m "生成されたメッセージ"
     ```

4. **リモートへのプッシュ**
   - 現在のブランチをリモートリポジトリにプッシュします。
     ```bash
     git push origin HEAD
     ```

5. **完了報告**
   - コミットメッセージの内容と、プッシュが完了したことをユーザーに報告します。

## 注意事項
- 意図しないファイルが含まれないよう、`.gitignore` が適切に設定されていることを前提とします。
- `git commit` が失敗した場合（差分がない場合など）は、適切にエラー処理を行いユーザーに通知します。
- `git push` 時の認証エラーなどは環境に依存するため、失敗時はログを確認しユーザーに伝えます。
