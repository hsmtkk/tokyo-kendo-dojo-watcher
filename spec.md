# 東京都剣道道場連盟サイト監視ツール 仕様書

東京都剣道道場連盟のウェブサイトを定期的に監視し、更新の有無をBandに通知する。

## 参照
- [Band Open API Documentation](https://developers.band.us/develop/guide/api)

## 監視対象
- URL: https://www.todoren.com/index.html

## 実行方式
- Google Apps Script (GAS)
- 実行頻度: 毎月1日、15日 (各日1回)

## 通知設計
- 通知先: Band (Open APIを使用)
- 通知タイミング:
  - 正常実行時: 更新の有無に関わらず、実行ごとに毎回通知
  - サイトエラー時: ターゲットサイトへのリクエスト失敗（エラー応答・タイムアウト）時に通知
- 通知形式: 新規投稿 (既存投稿への返信ではない)
- 通知メッセージ:
  - 更新ありの場合: 「東京都剣道道場連盟サイトに更新あり」
  - 更新なしの場合: 「東京都剣道道場連盟サイトに更新なし」
  - サイトエラーの場合: 「東京都剣道道場連盟サイトへのアクセスに失敗しました」
- 投稿内容: テキストのみのシンプルな投稿

## 更新の判定方法
- 判定基準: 応答HTTPヘッダーの `Last-Modified` または `ETag`
- 比較対象: 前回の実行時に保存した値
- 判定ロジック:
  - スクリプトプロパティに前回の値が保存されていない場合（初回実行時）は「更新なし」とみなし、現在の値を保存するのみとする
  - `Last-Modified` または `ETag` のいずれかに前回の値からの変更があれば「更新あり」とみなす
  - HTTPレスポンスボディは判定対象外とする

## データ永続化
- `PropertiesService` (スクリプトプロパティ) を使用して以下の値を保持する
  - 前回の `Last-Modified`
  - 前回の `ETag`
- クレデンシャル情報の管理:
  - `access_token`: Band APIのアクセストークン
  - `band_key`: 送信先Bandの識別子
  - これらはスクリプトプロパティに手動で設定し、ソースコードには記述しない

## エラーハンドリング
- サイトへのアクセス失敗時はBandにエラーメッセージを投稿する
- スクリプト自体の実行エラー（API制限など）発生時はログに記録する (Stackdriver Logging)
- トリガーの失敗通知設定により、エラー発生時は管理者へメール通知を行う

## デプロイ・運用
- デプロイツール: `clasp`
- 環境: ローカルまたは GitHub Actions
- トリガー設定: 
  - 1日と15日のそれぞれに対して、時間主導型トリガーを計2つ手動作成する
